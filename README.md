> Решение хакатона Норникеля (задача 3, микроскопия). Веб-приложение для сегментации
> минеральных фаз аншлифов: загрузка → нормализация → сегментация нейросетью → просмотр,
> ручная правка маски, competition-маска (обычные/тонкие срастания/тальк) с настройкой,
> экспорт в ZIP и PDF-отчёт по проекту.

---

## Что внутри

**Пайплайн:** фото → нормализация (`normalize_v4`, убирает цветовой каст и экспозицию) →
подгон под разрешение Qwen (одиночные — к ближайшему из 6 соотношений; панорамы — нарезка
на тайлы 2208×1656 с оверлапом) → сегментация **Qwen-Image-Edit-2511 + дообученная LoRA** через
fal.ai → постобработка (`clean_edges` + snap к палитре) → маска обратно в нативное разрешение.

**Фазы:** силикат, сульфиды, магнетит, тальк (модель сама определяет, какие есть на снимке).
**Competition-маска** (в превью): зелёный = обычные срастания, красный = тонкие, синий = тальк,
чёрный = силикат/магнетит. Тонкие/обычные разделяются по площади и толщине (площадь/периметр),
параметры настраиваются вживую и сохраняются на каждый файл.

**Возможности:** хаб проектов · загрузка одиночных/панорам с метаданными (µm/pixel, условия,
месторождение) · пакетная обработка · превью с настройкой · полноценный ручной редактор маски
(лассо/палочка/кисть/цветовой диапазон, привязка к палитре) · экспорт ZIP (маска/оригинал/наложение,
панорамы склеиваются) · PDF-отчёт по проекту с фазовым составом и классом руды.

## Стек
React + Vite + TypeScript (фронт) · Express + tsx (API) · Python (нормализация/сегментация/
постобработка/экспорт/отчёт: numpy, opencv, scipy, scikit-image, Pillow, fal-client, jinja2) ·
sharp (нарезка) · Caddy (reverse-proxy + TLS).

## Локальный запуск
```bash
cd app
npm install
# app/.env:  FAL_KEY=...   LORA_URL=<url лоры в fal storage>   PYTHON_BIN=python
npm run dev          # Vite :5555 + API :3001
```
Нужны Python-зависимости пайплайна (см. «Стек»). Секреты (`FAL_KEY`, `LORA_URL`) — только в
`app/.env`, в репозиторий не коммитятся.
